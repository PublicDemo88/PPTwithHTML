Fundamentos

EPAC en una frase

Enterprise Policy as Code consiste en definir y gobernar politicas de Azure como codigo versionado, validado y desplegable mediante CI/CD.

  • Politicas e iniciativas en ficheros
  • Control de cambios por pull request
  • Ejecucion automatizada en pipeline
  • Cumplimiento observable de forma continua

Pilares tecnicos

4
  • Definicion declarativa
  • Idempotencia y repetibilidad
  • Trazabilidad de cambios
  • Gobierno central con autonomia de equipos

Organizacion de ficheros

Estructura sugerida

epac-repo/ policies/ definitions/ initiatives/ assignments/ prod/ nonprod/ exemptions/ pipelines/ docs/

Buenas practicas

  • Separar definiciones de asignaciones por entorno.
  • Controlar excepciones como artefacto versionado.
  • Aplicar nomenclatura estable para scopes y owners.
  • Documentar criterios de severidad y remediacion.

Despliegue

Flujo tecnico de despliegue

1. PR

Cambio en politica o asignacion con revision de arquitectura/seguridad.

2. Validacion

Lint, schema checks, impacto y pruebas en entorno de validacion.

3. Release

Aplicacion automatizada en scopes objetivo (management group/subscription).

trigger: push/pull_request validate: policy syntax + assignment checks approve: security gate deploy: Azure Policy assignments report: compliance snapshot

Operacion diaria

Como se ve en Azure

  • Azure Policy: estado de cumplimiento por scope.
  • Microsoft Defender for Cloud: postura, recomendaciones y tendencias.
  • Dashboards para CISO, seguridad y equipos de plataforma.

Con una configuracion breve en los ficheros, los resultados quedan visibles de forma continua en portal Azure.

Roles y consumo

CISO / Seguridad

Seguimiento de postura y riesgo residual.

Plataforma

Remediacion y gestion de excepciones.

Equipos app

Feedback temprano de cumplimiento durante despliegues.
